## Codebase Patterns
- Prisma client is generated to `app/generated/prisma` and is gitignored; always run `bunx --bun prisma generate` after schema changes
- Use `@@map("snake_case")` for table names in Prisma models
- Use `@@index([userId])` on all user-owned models
- All user-owned models use `onDelete: Cascade` on the User relation
- Use String (not enum) for category fields; validate at application layer
- Typecheck: `bunx tsc --noEmit` (react-router typegen may not be needed for non-route changes)
- Database has drifted from migration history; `prisma migrate dev` won't work cleanly. Use `prisma db push` for schema sync if needed.
- HouseholdPreference compound unique: use `where: { userId_key: { userId, key } }` for upsert/findUnique
- VoltAgent tool pattern: `createTool({ name, description, parameters: z.object({}), execute: async (args, options) => {} })` with `options?.userId` for auth
- Tool upsert without compound unique: use `findFirst({ where: { userId, name } })` then branch to create/update
- Retriever pattern: extend `BaseRetriever`, super({ toolName, toolDescription }), implement `retrieve(input, options)` returning a string. Add to CombinedRetriever array in agents.ts.
- Sidebar nav links must be added in TWO places in app-layout.tsx: desktop sidebar (main nav) and mobile DrawerContent component
- Route layout files use `authMiddleware` export + `<Outlet />` -- child routes inherit auth
